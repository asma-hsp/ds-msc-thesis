\chapter{Methodology}\label{chapt:method}

This chapter presents the proposed approach to zero-shot instance-level
composed image retrieval. The central idea is to add a third retrieval
modality: in addition to the reference image and the modification text, a frozen
multimodal large language model generates a \emph{target caption} -- a short
description of the image the user has in mind -- and the three resulting
similarities are combined by a late-fusion product. The chapter is organised as
follows.
Section~\ref{sect:method_problem} formalises the composed image retrieval task
and fixes the notation used throughout the thesis.
Section~\ref{sect:method_dataset} describes the i-CIR benchmark.
Section~\ref{sect:method_eval} defines the evaluation protocol and metrics.
Section~\ref{sect:method_overview} gives an overview of the proposed retrieval
pipeline.
Section~\ref{sect:method_backbones} describes the vision--language backbones
used to embed the data.
Section~\ref{sect:method_basic} recalls the \textsc{Basic} baseline that this
work builds upon.
Section~\ref{sect:method_caption} introduces the target-caption generation
step, the central contribution.
Section~\ref{sect:method_fusion} presents the triplet late-fusion scoring
function.
Section~\ref{sect:method_impl} reports the implementation details.

\section{Problem Formulation and Notation}\label{sect:method_problem}
% =====================================================================

\paragraph{Composed image retrieval.}
A composed image retrieval query is a pair $q = (\qv, \qt)$, where $\qv$ is a
reference \emph{query image} and $\qt$ is a short \emph{query text} that
specifies how the desired result should differ from $\qv$. Given a database of
images $\DB = \{\xdb_1, \dots, \xdb_N\}$, the goal is to produce a ranking of
$\DB$ that places the image the user intends -- the result of applying the
modification $\qt$ to the instance shown in $\qv$ -- as close to the top as
possible. We denote by $\encv$ the visual encoder of a frozen vision--language
model and by $\enct$ its textual encoder; both map their input to an
$\ell_2$-normalised vector in a shared $d$-dimensional space. Image-to-image and
image-to-text similarities are then computed as the dot product of the
corresponding features.

\paragraph{Instance-level relevance.}
This thesis adopts the instance-level protocol of the i-CIR
benchmark~\cite{psomas2025icir}. Unlike category-level CIR, where any image of
the right category is acceptable, here a database image is \emph{relevant} to a
query only if it depicts the same object \emph{instance} as $\qv$, modified as
$\qt$ requests. Ground truth is defined at this instance granularity, so the
positive set of a query is typically small and specific. The benchmark
deliberately surrounds each query with \emph{hard negatives}, which we group
into three types following the dataset design: \emph{visual} hard negatives
(the same or a visually similar instance that does \emph{not} satisfy the
modification), \emph{textual} hard negatives (images that satisfy the
modification text but show a \emph{different} instance), and \emph{composed}
hard negatives (the correct instance under a \emph{different} modification).
This structure is what makes instance-level CIR difficult: a method must
jointly preserve the visual identity of the instance \emph{and} apply the
precise change demanded by the text, since collapsing onto either modality
alone aligns the method with one of the hard-negative types.

\paragraph{Zero-shot assumption.}
We operate strictly in the \emph{zero-shot} regime: no component of the system
is trained or fine-tuned on i-CIR. All encoders $\encv, \enct$ are
off-the-shelf, frozen vision--language models, the caption generator is a
frozen MLLM, and every fusion and post-processing step is training-free. We
adopt this setting because the zero-shot regime is the more practical and
scalable one for composed image retrieval in general:
\begin{itemize}
  \item \textbf{Supervised training is expensive and labour-intensive.} It
        requires large-scale annotated triplets of reference image, modification
        text, and target image, which are costly to collect.
  \item \textbf{It does not scale.} Building a dedicated training set for every
        possible kind of modification a user might request is unrealistic.
  \item \textbf{Trained models generalise poorly.} A fusion model trained on one
        CIR dataset often transfers badly to another.
\end{itemize}
Operating zero-shot side-steps all three issues; on i-CIR specifically, it
additionally isolates the contribution of the proposed retrieval signal from any
benefit of supervision.

\section{The i-CIR Dataset}\label{sect:method_dataset}
% =====================================================================

We conduct all experiments on i-CIR~\cite{psomas2025icir}, the instance-level
composed image retrieval benchmark. The benchmark is organised around object
\emph{instances}: each instance is associated with one or more composed queries,
where a query pairs a reference image $\qv$ of the instance with a modification
text $\qt$ describing a desired change. Relevance is defined at instance
granularity, exactly as in Section~\ref{sect:method_problem}.

\paragraph{Query-specific instance pool.}
Rather than ranking each query against the full database, i-CIR equips every
instance with a dedicated \emph{pool} of candidate images. This pool gathers the
positive (relevant) images of all queries belonging to the instance together
with a curated set of hard negatives of the three types introduced in
Section~\ref{sect:method_problem}. Retrieval and evaluation for a query are
carried out within this pool, which concentrates closely related distractors
around each query and makes the ranking task substantially harder than retrieval
against an unconstrained gallery.

\paragraph{Scale.}
% HIGHLIGHT: verify these counts against the i-CIR data card.
The full benchmark comprises $1{,}883$ queries and $N = 752{,}092$ database
images. For development we use a fixed $20\%$ instance subset ($400$ queries and
$N = 143{,}769$ database images; see Section~\ref{sect:method_impl}), on which
the ablation studies of Chapter~\ref{chapt:results} are conducted.

\section{Evaluation Protocol and Metrics}\label{sect:method_eval}
% =====================================================================

\paragraph{Ranking protocol.}
Following the i-CIR protocol~\cite{psomas2025icir}, each query is evaluated
within its query-specific instance pool (Section~\ref{sect:method_dataset}): the
pool is ranked by decreasing fused similarity and the ranking is compared against
the query's relevant set.

\paragraph{Precision and recall.}
Given a ranking, let $\mathrm{rel}(j)\in\{0,1\}$ indicate whether the item at
rank $j$ is relevant, and let $R$ be the total number of relevant items for the
query. At a rank cutoff $k$, precision and recall are
\begin{equation}\label{eq:prec_rec}
    \mathrm{P}@k = \frac{1}{k}\sum_{j=1}^{k}\mathrm{rel}(j),
    \qquad
    \mathrm{R}@k = \frac{1}{R}\sum_{j=1}^{k}\mathrm{rel}(j),
\end{equation}
i.e. the fraction of the top-$k$ that is relevant, and the fraction of all
relevant items recovered within the top-$k$, respectively.

\paragraph{Average precision.}
Average precision (AP) summarises the whole precision--recall curve into a single
number. Sweeping the cutoff $k$ traces the sequence of points
$(\mathrm{R}@k, \mathrm{P}@k)$, and AP is the area under this curve. We use the
trapezoidal (Oxford-style) estimate adopted by the benchmark and classical
instance retrieval~\cite{philbin2007oxford},
\begin{equation}\label{eq:ap}
    \mathrm{AP} = \sum_{i \geq 1} (r_i - r_{i-1})\,
    \frac{p_i + p_{i-1}}{2},
\end{equation}
where $(r_i, p_i)$ are successive recall--precision points. AP is high only when
relevant items are concentrated near the top of the ranking.

\paragraph{Mean average precision.}
% HIGHLIGHT: confirm this matches the paper's macro/micro definition exactly.
The mean average precision (mAP) is AP averaged over a set of queries. Because
the number of queries per instance is uneven, two averagings are distinguished.
The \emph{micro}-averaged mAP averages AP over \emph{all queries} equally,
\begin{equation}\label{eq:micro_map}
    \textsc{mAP} = \frac{1}{|Q|}\sum_{q\in Q}\mathrm{AP}(q),
\end{equation}
where $Q$ is the set of all queries. The \emph{macro}-averaged mAP first
averages AP within each instance and then averages across instances,
\begin{equation}\label{eq:macro_map}
    \textsc{mmAP} = \frac{1}{|\mathcal{I}|}\sum_{I\in\mathcal{I}}
    \frac{1}{|Q_I|}\sum_{q\in Q_I}\mathrm{AP}(q),
\end{equation}
where $\mathcal{I}$ is the set of instances and $Q_I$ the queries of instance
$I$. Macro-averaging gives every instance equal weight regardless of how many
queries it has, whereas micro-averaging gives every query equal weight.
Following the i-CIR benchmark, \textbf{macro-mAP is the primary metric}; it is
the headline number in all tables unless stated otherwise.

\paragraph{Auxiliary metrics.}
We additionally report rank-cutoff metrics that characterise the top of the
ranking, which is what a user sees first: $\mathrm{R}@k$ of
Equation~\eqref{eq:prec_rec} for $k \in \{10, 100\}$, and $\mathrm{Hit}@k$
(whether at least one relevant item appears in the top-$k$) for
$k \in \{1, 10\}$.

\section{Pipeline Overview}\label{sect:method_overview}
% =====================================================================

Figure~\ref{fig:pipeline} summarises the proposed pipeline. The system has three
parallel branches that each produce a similarity between the query and a
candidate database image $\xdb$:
\begin{enumerate}
  \item the \emph{image branch} encodes the reference image $\qv$ with $\encv$
        and compares it to the encoded candidate, giving
        $\sv = \simop(\encv(\qv), \encv(\xdb))$;
  \item the \emph{text branch} encodes the modification text $\qt$ with $\enct$
        and compares it to the candidate, giving
        $\st = \simop(\enct(\qt), \encv(\xdb))$;
  \item the \emph{caption branch}, the contribution of this thesis, generates a
        target caption $\qc$ with a frozen MLLM, encodes it with $\enct$, and
        compares it to the candidate, giving
        $\sc = \simop(\enct(\qc), \encv(\xdb))$.
\end{enumerate}
The three scores are combined by the late-fusion product of
Section~\ref{sect:method_fusion} into a single score per candidate, and the
database is ranked by this fused score. Two properties are worth emphasising.
First, \emph{all encoders are frozen and off-the-shelf}: every model is used
exactly as released, with no training or fine-tuning on the retrieval task.
Second, the
caption branch is purely \emph{additive} -- it supplies a strong text-side query
without removing the visual signal from the image branch, which is what allows
the fusion to respect instance identity while applying the modification.

\section{Vision--Language Backbones}\label{sect:method_backbones}
% =====================================================================

The encoders $\encv$ and $\enct$ are supplied by a frozen, contrastively
pretrained vision--language model that maps images and text into a shared
embedding space. We evaluate the pipeline with four such backbones spanning two
model families and two capacity levels (Table~\ref{tab:backbones}), both to
measure how much the choice of backbone matters and to report numbers comparable
to the i-CIR baseline.

The first family is CLIP~\cite{radford2021clip}, which learns a joint
image--text space by contrastive training on large image--caption datasets: a
symmetric InfoNCE objective pulls matching image--text pairs together and pushes
mismatched pairs apart within each batch. We use two CLIP variants: ViT-L/14,
the backbone most commonly reported in the zero-shot CIR literature, which makes
our numbers directly comparable; and the higher-capacity ViT-H/14 from
OpenCLIP~\cite{ilharco2021openclip}, the strongest CLIP model on the duplet
baseline.

The second family is SigLIP~\cite{zhai2023siglip}, which replaces CLIP's
softmax-based contrastive loss with a pairwise \emph{sigmoid} loss: each
image--text pair is scored independently as a binary matched/not-matched
decision, removing the dependence on a global softmax over the batch. This
stabilises training, scales better to large batch sizes, and yields stronger
image--text alignment. We use SigLIP ViT-L/16 at $256$px resolution and the more
recent, higher-capacity SigLIP2 g/16 at $384$px~\cite{tschannen2025siglip2},
which is the best-aligned encoder of the four and, as
Chapter~\ref{chapt:results} shows, the strongest backbone for our pipeline.

For every backbone, the reference image $\qv$ and all database images $\xdb$ are
embedded with the visual encoder $\encv$, while the modification text $\qt$ and
the generated caption $\qc$ are embedded with the textual encoder $\enct$ into
the same space; all embeddings are $\ell_2$-normalised. No backbone is
fine-tuned -- each is used exactly as released -- which keeps the comparison fair
and the pipeline zero-shot.

\section{The \textsc{Basic} Baseline}\label{sect:method_basic}
% =====================================================================

The method of this thesis builds on \textsc{Basic}, the training-free zero-shot
pipeline introduced together with the i-CIR benchmark~\cite{psomas2025icir} and
re-implemented here. \textsc{Basic} rests on the assumption that the two
modalities of a composed query encode \emph{complementary} information that
jointly defines the retrieval target, which makes composed retrieval analogous to
a logical conjunction over the modalities: the result sought is one that is
simultaneously relevant to the reference image \emph{and} to the modification
text.\footnote{This assumption holds well for standard composite queries (e.g.\
\emph{this image} and the concept ``winter''), but may weaken when one modality
dominates (e.g.\ purely textual transformations) or when the text query is highly
entangled with the image content, as in CIRR-like datasets; in such cases the
benefit of the mechanisms below diminishes.} Following this assumption,
\textsc{Basic} computes a similarity per modality and fuses the two. Before
fusing, it improves each modality by removing modality-specific noise and
spurious correlations that interfere with their combination: visual features can
be entangled with background clutter, image composition, or dataset-specific
style, while textual features can reflect lexical biases or corpus-level drift.
The pipeline is a sequence of conceptually simple yet effective steps that
progressively filter out these artefacts. We summarise them below, marking which
parts this thesis keeps, tunes, or supersedes; the empirical contribution of each
is quantified in Chapter~\ref{chapt:results}.

\paragraph{Centering for bias removal.} Each embedding is centered by subtracting
a precomputed modality mean, which captures low-level regularities (general
visual patterns for images, common linguistic patterns for text) and whose
removal isolates semantic content from distributional bias
(Equation~\eqref{eq:centering}). A mean $\boldsymbol{\mu}^v$ is subtracted from
the image features $\encv(\qv)$ and from the database features, and a text mean
$\boldsymbol{\mu}^t$ from the text features $\enct(\qt)$. For scale and
generalisation, $\boldsymbol{\mu}^v$ is estimated on a large external dataset
such as LAION~\cite{schuhmann2022laion5b} and $\boldsymbol{\mu}^t$ on a
predefined textual corpus. The choice of image mean is studied in
Section~\ref{subsect:method_centering}.

\paragraph{Projection onto a semantic subspace.} To retain information about the
main object while suppressing style, object domain, and background -- exactly the
dimensions that common text modifications target -- \textsc{Basic} projects the
centered image features onto a low-dimensional subspace derived from text
features. Two textual corpora are used: an \emph{object} corpus $\mathcal{C}^+$
of object-oriented terms (e.g.\ ``building'', ``dog'') and a \emph{stylistic}
corpus $\mathcal{C}^-$ of style, viewing-condition, and context terms (e.g.\
``cartoon'', ``aerial view'', ``in a cloudy day''). A weighted contrastive
covariance matrix
\begin{equation}\label{eq:basic_proj}
  C = (1-\alpha)\,C^{+} - \alpha\,C^{-},
  \qquad
  C^{\pm} = \frac{1}{|\mathcal{C}^{\pm}|}\sum_{x\in\mathcal{C}^{\pm}}
  \big(\enct(x)-\boldsymbol{\mu}^t\big)\big(\enct(x)-\boldsymbol{\mu}^t\big)^{\!\top}
\end{equation}
is formed, with $\alpha$ an empirically chosen hyperparameter. The top-$k$
eigenvectors of $C\in\mathbb{R}^{d\times d}$ form a projection matrix
$P\in\mathbb{R}^{d\times k}$ that captures directions of high variance in
$\mathcal{C}^+$ and low variance in $\mathcal{C}^-$, emphasising object-specific
cues while suppressing style-related variation; centered image features
$\bar{x}^v$ are projected as $P^{\top}\bar{x}^v\in\mathbb{R}^{k}$. The corpora
need not match the retrieval domain -- even generic terms (e.g.\ ImageNet class
names) transfer well. This is the step our high-level overview earlier called
dimensionality reduction; it is in fact a \emph{semantic} projection rather than
plain PCA.

\paragraph{Image query expansion.} Optionally, the image-query feature is
enhanced by a weighted combination of the top-ranked database features it
retrieves, with weights increasing in the corresponding similarities -- a
standard recall-boosting technique in instance retrieval~\cite{philbin2007oxford}.

\paragraph{Contextualization of text queries.} Because CLIP is trained on full
captions, bare single-word or fragmentary text queries (e.g.\ ``sculpture'',
``during sunset'') are out-of-distribution and produce poorly aligned text
features. \textsc{Basic} therefore \emph{contextualizes} $\qt$ into caption-like
phrases by inserting random terms from the object corpus $\mathcal{C}^+$ before
and after it (e.g.\ ``dog during the sunset'', ``sculpture dog''), embedding all
variants, centering them, and averaging the result into a single, more robust
text feature. This is the step that the target caption of
Section~\ref{sect:method_caption} is designed to replace with a stronger,
self-contained textual signal.

\paragraph{Score normalisation and fusion.} For a database image $\xdb$ with
centered feature $\bar{x}^v$, and given the centered image query $\bar{q}^v$ and
the contextualized centered text query $\bar{q}^t$, the per-modality similarities
are
\begin{equation}\label{eq:basic_scores}
  \sv = \big\langle P^{\top}\bar{x}^v,\; P^{\top}\bar{q}^v\big\rangle,
  \qquad
  \st = \big\langle \bar{x}^v,\; \bar{q}^t\big\rangle ,
\end{equation}
and are fused multiplicatively, $s = \sv\,\st$, reflecting the \textsc{and}
assumption. To correct the range imbalance between modalities, each similarity
is affinely rescaled using its empirical minimum $s_{\min}<0$ (estimated on an
external dataset), so that $s_{\min}\!\mapsto\!0$ and $0\!\mapsto\!1$:
\begin{equation}\label{eq:basic_minnorm}
  \tilde{s} = \frac{s - s_{\min}}{|s_{\min}|}.
\end{equation}
The normalised scores are finally fused with a regulariser inspired by the
Harris corner detector~\cite{harris1988corner},
\begin{equation}\label{eq:basic_harris}
  \tilde{s}^{f} = \tilde{s}^{v}\,\tilde{s}^{t}
  - \lambda\big(\tilde{s}^{v} + \tilde{s}^{t}\big)^{2},
\end{equation}
whose first term rewards items jointly relevant to both modalities and whose
second term penalises items activated by only one modality, thereby suppressing
single-modality false positives; the scalar $\lambda$ is fixed across all
experiments.

\paragraph{Efficiency.} The entire pipeline is (deep-network) training-free and
scales near-linearly with the database, since scoring reduces to inner products
that off-the-shelf similarity-search libraries handle efficiently. Moreover,
centering and projection can be folded into the query side and applied on the fly
over a stored index of \emph{raw} features, so the mean and the projection matrix
can be swapped -- for example with domain-specific knowledge -- without touching
the stored index, which makes \textsc{Basic} well-suited to large-scale
deployment with no adaptation, fine-tuning, or backpropagation.

\subsection{Feature Centering}\label{subsect:method_centering}
Centering subtracts a fixed mean $\mu$ from each embedding and re-normalises, as
in Equation~\eqref{eq:centering}. The mean can be estimated from the database
itself (the \emph{global} mean) or from an external corpus -- here a one-million
image sample of LAION~\cite{schuhmann2022laion5b} (the \emph{LAION-1M} mean).
Both choices remove the embedding-space bias that otherwise inflates all
pairwise similarities; whether the source of the mean matters in practice is a
question we answer empirically in Section~\ref{sect:res_centering}.

\subsection{Duplet Baseline Score}\label{subsect:method_duplet}
Throughout this thesis we refer to the two-modality \textsc{Basic} score --
image fused with (contextualized) text -- as the \emph{duplet} baseline, against
which the triplet method of Section~\ref{sect:method_fusion} is compared. For
consistency with the triplet fusion, we instantiate \textsc{Basic}'s
multiplicative \textsc{and} (Equation~\eqref{eq:basic_harris}) in the simpler
clamped-product form
\begin{equation}\label{eq:duplet}
    s_{\text{duplet}}(q, \xdb) =
    \clamp(\st)\cdot\clamp(\sv),
\end{equation}
where $\clamp(\cdot)$ maps negative similarities to zero
(Section~\ref{sect:method_fusion}). The triplet score of this thesis extends
Equation~\eqref{eq:duplet} with the caption term.

\section{Target-Caption Generation}\label{sect:method_caption}
% =====================================================================

This section describes the central contribution: generating, for each query, a
caption of the \emph{imagined target} image and using its embedding as a third
retrieval signal.

% TODO: Add a paragraph (before or after the Idea paragraph) justifying the
% choice of InternVL-3.5-8B: open-source, strong multimodal instruction-following,
% high-resolution tiling, practical size (8B) for cluster inference.
\paragraph{Idea.} The modification text $\qt$ is a \emph{relative} instruction
and a weak standalone query: a phrase such as ``make it red'' is meaningless to
a text encoder without knowing what should become red. We instead give a frozen
multimodal large language model, InternVL-3.5-8B~\cite{wang2025internvl35}, the
reference image $\qv$ and the modification $\qt$ \emph{together}, and prompt it
to imagine the modification applied and to emit a short, retrieval-oriented
description of the intended target. This caption $\qc$ ``applies'' the edit in
language space, turning an implicit cross-modal composition into an explicit,
self-contained text query that the frozen text encoder $\enct$ embeds well.

\paragraph{Output contract.} Every instruction asks the model for a structured
output: a first line containing a JSON object with instance metadata (whether
the instance is identifiable, its name, type, attributes, and the modification),
followed by a line beginning exactly with \texttt{@Final Caption:} that contains
the caption. The caption text after this marker is parsed out and used as $\qc$;
the JSON line serves only to steer the model's reasoning. The captions are
constrained to focus on the \emph{single} main instance the modification refers
to, to suppress background and style, to name a well-known entity only at high
confidence (otherwise to list distinctive attributes for retrieval), and to be
concise and factual.

\paragraph{Instruction design.} A short, retrieval-oriented description of an
imagined image is sensitive to how the model is prompted. We therefore designed
a space of five instruction variants, summarised in
Table~\ref{tab:instructions}, that share a common contract -- recognise the
single main instance, apply the modification, and emit a concise caption free of
scene and aesthetic noise -- but differ in how they enforce it.
\textsc{F} (\emph{adaptive precision}) gates instance recognition on a
confidence threshold, naming a well-known entity only when highly confident and
otherwise falling back to distinctive visual traits.
\textsc{G} (\emph{constraint-priority}) first decomposes the modification into
atomic constraints to guarantee full coverage, then binds them to the instance
identity.
\textsc{H} (\emph{anti-noise contract}) enforces an explicit quality contract
with a forbidden-vocabulary list that blocks background, lighting, and aesthetic
descriptors.
\textsc{I} (\emph{slot-fill template}) imposes a rigid
``[instance phrase] [modification phrase]'' template that suppresses noise
structurally.
\textsc{J} (\emph{draft--refine}) drafts a caption and then silently self-checks
it against the identity, constraint-coverage, noise, and length requirements
before emitting a refined version.
The per-instruction comparison is reported in
Section~\ref{subsect:res_ablation_instr}.

\paragraph{Why these five.} The variants were not chosen arbitrarily: they were
designed from an analysis of an earlier, larger pool of prompts evaluated on the
development subset, which exposed three recurring failure modes. Concise,
step-by-step prompts recognised well-known named entities accurately but produced
captions that were \emph{too sparse} for unrecognisable instances (e.g.\
collapsing a specific bridge to ``a bridge''); richer, more descriptive prompts
improved coverage but injected \emph{scene and aesthetic noise} (lighting,
background, mood); and prompts that decomposed the modification into atomic parts
ensured \emph{constraint coverage} but often lost the instance's visual identity.
Each of the five variants in Table~\ref{tab:instructions} is a targeted remedy
that keeps the conciseness that works for named entities, forces richer
distinctive traits for unnamed ones, requires every modification constraint to
appear, and adds explicit anti-patterns that block scene and aesthetic noise.
They differ only in \emph{how} they enforce this shared contract -- confidence
gating (F), constraint-first decomposition (G), an explicit forbidden-vocabulary
contract (H), a rigid fill-in template (I), or an internal draft-and-self-check
loop (J) -- so that embedding-space averaging over them (\texttt{avg-5}) can
exploit their complementary strengths. The verbatim prompts are reproduced in
Appendix~\ref{app:prompts}.

\paragraph{Robustness via averaging.} Rather than commit to a single
instruction, we reduce caption variance by \emph{averaging in embedding space}.
Let $\qc^{(1)}, \dots, \qc^{(5)}$ be the captions produced by the five
instructions for a given query. The default caption embedding, denoted
\texttt{avg-5}, is the $\ell_2$-normalised mean of their individual text
embeddings,
\begin{equation}\label{eq:avg5}
    \enct(\qc) \;=\; \frac{\bar{e}}{\lVert \bar{e}\rVert},
    \qquad
    \bar{e} \;=\; \frac{1}{5}\sum_{j=1}^{5}\enct\!\big(\qc^{(j)}\big).
\end{equation}
Averaging the embeddings (not the texts) keeps the signal that is consistent
across instructions and attenuates the idiosyncrasies of any single prompt. We
use \texttt{avg-5} as the default and compare it against the best single
instruction in Chapter~\ref{chapt:results}.

\section{Triplet Late-Fusion Scoring}\label{sect:method_fusion}
% =====================================================================

The three branches of Section~\ref{sect:method_overview} produce, for a query
$q = (\qv, \qt)$ with generated caption $\qc$ and a candidate database image
$\xdb$, three cosine similarities -- the \emph{image similarity} $\sv$, the
\emph{text similarity} $\st$, and the \emph{caption similarity} $\sc$ --
defined as
\begin{align}
  \sv &= \simop\!\big(\encv(\qv),\, \encv(\xdb)\big),
        && \text{(image similarity)} \label{eq:s_img}\\
  \st &= \simop\!\big(\enct(\qt),\, \encv(\xdb)\big),
        && \text{(text similarity)} \label{eq:s_txt}\\
  \sc &= \simop\!\big(\enct(\qc),\, \encv(\xdb)\big).
        && \text{(caption similarity)} \label{eq:s_cap}
\end{align}
We fuse them by the \emph{clamped triplet product}
\begin{equation}\label{eq:triplet}
    s_{\text{triplet}}(q, \xdb) =
    \clamp\!\big(\sc\big)\cdot
    \clamp\!\big(\st\big)\cdot
    \clamp\!\big(\sv\big),
    \qquad
    \clamp(s) = \max(s, 0),
\end{equation}
and rank the database by $s_{\text{triplet}}$.

\paragraph{Why a product.} The product behaves as a soft logical
\textsc{and}: a candidate scores highly only if it agrees with \emph{all three}
signals -- it must look like the same instance ($\sv$), be consistent with the
modification text ($\st$), and match the imagined-target description ($\sc$). A
candidate that satisfies only one or two terms is heavily down-weighted, which
is exactly the behaviour needed against the instance-level hard negatives of
Section~\ref{sect:method_problem}: visual negatives fail $\st$ and $\sc$, while
textual negatives fail $\sv$. A sum, by contrast, would let one strong term mask
the failure of another and would reward the very hard negatives the benchmark is
designed around.

\paragraph{Why clamping.} Cosine similarities can be negative. Without
clamping, the product of two negative terms would become a large positive score
-- spuriously promoting candidates that are \emph{dissimilar} on two branches --
and a single negative term could flip the sign of the whole score. Clamping each
similarity to $[0, \infty)$ before multiplying removes these sign artefacts and
keeps the product monotone in each branch.

\paragraph{Relation to the duplet baseline.} Dropping the caption term recovers
the duplet baseline of Equation~\eqref{eq:duplet}. The triplet score is thus a
strict, additive extension of \textsc{Basic}'s fusion by the proposed third
modality; Chapter~\ref{chapt:results} measures the gain this term provides.

\section{Implementation Details}\label{sect:method_impl}
% =====================================================================

\paragraph{Software and hardware.} The pipeline is implemented in Python with
PyTorch. Contrastive backbones are loaded through OpenCLIP and HuggingFace
Transformers, and the InternVL caption generator runs through HuggingFace
Transformers. All large-scale embedding and retrieval jobs run on a SLURM-managed
HPC cluster, distributed across whichever NVIDIA GPU nodes were available for a
given job depending on the task, node availability, and the memory it required;
the heavier caption-generation and full-database embedding jobs were scheduled on
the higher-memory nodes.

\paragraph{Caption generation settings.} Captions are generated with
InternVL-3.5-8B using its high-resolution tiling input and a fixed decoding
configuration shared across all five instructions.

\paragraph{Embedding the full database.} Embedding the full $752$K-image
database is the most expensive step. It is performed once per backbone in a
\emph{sharded, checkpointed} manner: the database is split into shards, each
shard is embedded independently and written to disk, and a job that is preempted
or fails resumes from the last completed shard. Query images, query texts,
captions, and the contextualized-text corpora are embedded the same way.
Embeddings are cached on disk and reused across all retrieval configurations, so
that the expensive encoding is never repeated when only the fusion or
post-processing changes.

\paragraph{Development subset.} To iterate quickly, ablations are run on a fixed
$20\%$ instance subset (\texttt{instance\_sample20\_seed42}: $400$ queries and
$143{,}769$ database images). The subset is sampled by instance with a fixed
seed so that it is reproducible and representative. We treat it as a fair
development proxy because the subset duplet baseline closely matches the full
baseline (the CLIP-L text$\times$image macro-mAP is within a fraction of a point
of the paper's full-dataset baseline), so trends observed on the subset are
expected to hold at full scale; the headline numbers in Chapter~\ref{chapt:results}
are nonetheless always reported on the full dataset.

\paragraph{Result storage and reproducibility.} Retrieval rankings and the
associated evaluation artefacts are stored in a compact, sparse (CSR-packed)
format, alongside the configuration that produced them. All runs are driven by
explicit configuration files and fixed random seeds, so that every reported
number can be regenerated from the stored embeddings and configs.
